def entropy(signal):
        '''
        function returns entropy of a signal
        signal must be a 1-D numpy array
        '''
        lensig=signal.size
        symset=list(set(signal))
        propab=[np.size(signal[signal==i])/(1.0*lensig) for i in symset]
        ent=np.sum([p*np.log2(1.0/p) for p in propab])
        return ent

def entropy_heatmap(image, k=5):
    '''
    image: 2d numpy array as an input
    k (int) : number of neighbours
    '''
    S=image.shape
    E=np.array(image)
    for row in range(S[0]):
        for col in range(S[1]):
            Lx=np.max([0,col-k])
            Ux=np.min([S[1],col+k+1]) # +1 due to x[a:b] -> [a;b)
            Ly=np.max([0,row-k])
            Uy=np.min([S[0],row+k+1])
            region=image[Ly:Uy,Lx:Ux].flatten()
            E[row,col]=entropy(region)
    return E

def digitize(image, bins=10):
    _, bin_edges = np.histogram([0,255], bins=bins)
    digitized = np.digitize(image, bin_edges)
    return digitized

Z analizy heat map wynika, że największa różnica jest w obszarach gdzie kolor jest mniej więcej stały, domyślam się że HSV nieco lepiej radzi sobie z szumem, ponieważ w RGB każda warstwa posiada stałą cząstke szumu niezależnie od koloru, a jeżeli mamy np. zieloną trawę to warstwa H w HSV prawdopodobnie ma bardzo niską entropię - co nie występuje w RGB ponieważ warstwa G będzie miała też elementy szumu pochodzące z różnic w jasności pikseli. Podobnie np. woda lub mgła na warstwie odpowiedzialnej za jasność będzie miała niską entropię. Z analizy rozkładów poszczególnych entropii w zależności od ilości sąsiadów widzimy, że rośnie ona gdy ilość sąsiadów jest większa w podobny sposób zarówno dla RGB jak i HSV. Mediana ze średnich entropii jest zawsze mniejsza dla HSV.
